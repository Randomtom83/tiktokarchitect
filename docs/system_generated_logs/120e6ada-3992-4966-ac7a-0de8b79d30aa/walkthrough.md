# Walkthrough — Desktop Dashboard AJAX Overhaul & Daily One Thing Repetition Fix

We have completed the desktop dashboard AJAX overhaul and successfully resolved the bug causing duplicate Daily One Thing focus items to persist on the UI.

---

## 1. Summary of Changes

### A. Dashboard AJAX Overhaul
- **POST Controller Dual-Mode:** Configured the POST handler in [dashboard.php](file:///Y:/greenstories/TomOnly/lifeoftom/dashboard.php) to output JSON results for `fetch()` requests and exit immediately, bypassing full-page reloads.
- **Glassmorphic Toasts:** Injected styles and a JS `showToast` method to display smooth, visually appealing toast confirmations.
- **In-Place Drawer Updates:** Updated the drawer code to support a `preserveTab` flag. Saving state, memory facts, guidelines, or toggling coach activity now refreshes the drawer in-place without resetting the tab index or chart.
- **Form Submissions:** Intercepted all forms globally. Deletions confirm first, and marking inbox items done or dismissing patterns now prompts for optional notes that are returned to the database.
- **Live Card Syncing:** Created updates that sync coach stats, rolling summaries, open threads, and paused banners directly on the dashboard's main page cards without a page refresh.

### B. Daily One Thing Repetition Fix
- **System Prompt Guardrail:** Added a hard instruction to [orchestrator.md](file:///Y:/greenstories/TomOnly/lifeoftom/content/pages/agents/orchestrator.md) warning the orchestrator meta-agent to never repeat yesterday's action verbatim or near-verbatim.
- **Programmatic Duplicate Filter:** Injected code in [orchestrator.py](file:///Y:/greenstories/TomOnly/lifeoftom/engine/orchestrator.py) that queries the last 3 days of focus items and automatically rejects any verbatim duplicates generated by the local model.
- **UI & API Carry-over Rules:** Modified `dashboard.php` and [api.php](file:///Y:/greenstories/TomOnly/lifeoftom/api.php) to only display the latest focus item if it matches today's date, or if it is from yesterday and remains pending (feedback is null).
- **Orchestrator Run Status:** If today's focus is empty, the dashboard now checks if today's run has completed. It shows a helpful message depending on whether the orchestrator decided there was "No synthesis today" or if today's batch is still pending.
